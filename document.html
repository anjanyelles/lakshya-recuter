1. Introduction

1.1 Purpose

Build a modern, AI-powered Applicant Tracking System that allows companies to:

  • Post job openings
  • Receive and manage applications
  • Screen candidates using AI automation
  • Schedule and conduct interviews
  • Make data-driven hiring decisions
  • Track entire recruitment lifecycle

1.2 Scope

1.2.1 In Scope

  • Complete recruitment workflow automation
  • Multi-role access (Candidate, Recruiter, Hiring Manager, Admin)
  • AI-powered resume parsing and candidate matching
  • Interview scheduling and management
  • Email and notification system
  • Analytics and reporting dashboard
  • Document management (resume, cover letters)
  • Real-time status tracking
  • Mobile-responsive web application

1.2.2 Out of Scope (Future Phases)

  • Video interviewing platform (integration only)
  • Background verification services
  • Payroll integration
  • Employee onboarding
  • Native mobile apps (iOS/Android)
  • Multi-language support

2. System Overview

2.1 Product Perspective

The ATS platform is a standalone, cloud-based SaaS application that operates independently but can integrate with:

  • Email services (SMTP, SendGrid, AWS SES)
  • Calendar systems (Google Calendar, Outlook)
  • Video conferencing (Zoom, Google Meet)
  • Cloud storage (AWS S3, Azure Blob Storage)
  • AI services (OpenAI, Anthropic Claude)

2.3 User Characteristics

2.3.1 Technical Skill Level

  • Candidates: Basic computer literacy, familiar with job portals
  • Recruiters: Intermediate, familiar with HR software
  • Hiring Managers: Basic to intermediate
  • Admins: Advanced, technical background required

2.3.2 Domain Knowledge

  • Recruiters and hiring managers have deep understanding of recruitment process
  • System should guide candidates through application process
  • No technical training required for daily operations

2.4 Operating Environment

2.4.1 Client Side

  • Web Browsers: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
  • Devices: Desktop, laptop, tablet (responsive design)
  • Internet: Minimum 2 Mbps connection
  • Screen Resolution: Minimum 1024x768

2.4.2 Server Side

  • Cloud Platform: AWS/Azure/GCP
  • Operating System: Linux (Ubuntu 20.04 LTS or higher)
  • Database: PostgreSQL 13+, Redis 6+
  • Application Server: Node.js runtime
  • Web Server: Nginx/Apache (reverse proxy)

3. Functional Requirements

3.1 User Management Module

FR-UM-001
HIGH

User Registration

User Role: All

Description: Allow users to create accounts on the platform.

Requirements:
FR-UM-001.1: System shall provide registration form with fields:
  • Email address (unique, validated)
  • Password (minimum 8 characters, must contain uppercase, lowercase, number, special character)
  • First name
  • Last name
  • User role (Candidate/Recruiter)
  • Phone number (optional)
FR-UM-001.2: System shall validate email format in real-time
FR-UM-001.3: System shall check if email already exists before registration
FR-UM-001.4: System shall hash password before storing in database
FR-UM-001.5: System shall send verification email with activation link
FR-UM-001.6: System shall activate account only after email verification
FR-UM-001.7: Verification link shall expire after 24 hours
FR-UM-001.8: System shall display appropriate success/error messages
Acceptance Criteria:
  • User can successfully register with valid details
  • Duplicate email registration is prevented
  • Password is stored encrypted
  • Verification email is sent within 30 seconds
  • User cannot login before email verification
FR-UM-002
HIGH

User Authentication

User Role: All

Description: Secure login mechanism for all users.

Requirements:
FR-UM-002.1: System shall provide login form with:
  • Email address field
  • Password field
  • "Remember me" checkbox
  • "Forgot password" link
FR-UM-002.2: System shall validate credentials against database
FR-UM-002.3: System shall generate JWT token upon successful login
FR-UM-002.4: JWT token shall contain:
  • User ID
  • User role
  • Expiration timestamp
FR-UM-002.5: Token expiration:
  • Access token: 15 minutes
  • Refresh token: 7 days
FR-UM-002.6: System shall implement refresh token mechanism
FR-UM-002.7: System shall log all login attempts with:
  • Timestamp
  • IP address
  • User agent
  • Success/failure status
FR-UM-002.8: System shall lock account after 5 failed login attempts
FR-UM-002.9: Account unlock options:
  • Automatic unlock after 30 minutes
  • Email verification link
FR-UM-002.10: System shall support "Remember me" for 30 days
Acceptance Criteria:
  • Valid users can login successfully
  • Invalid credentials show appropriate error
  • Token expires correctly
  • Failed attempts are tracked
  • Account locks after 5 failures
FR-UM-003
HIGH

Password Management

User Role: All

Description: Password reset and change functionality.

Requirements:
FR-UM-003.1: Forgot Password Flow:
  • User enters email address
  • System sends password reset link
  • Link expires in 1 hour
  • User creates new password
  • System confirms password change
FR-UM-003.2: Change Password (Logged-in Users):
  • User enters current password
  • User enters new password
  • User confirms new password
  • System validates current password
  • System updates password
FR-UM-003.3: Password validation rules:
  • Minimum 8 characters
  • At least 1 uppercase letter
  • At least 1 lowercase letter
  • At least 1 number
  • At least 1 special character (!@#$%^&*)
  • Cannot be same as previous 3 passwords
FR-UM-003.4: System shall show password strength indicator
FR-UM-003.5: System shall send confirmation email after password change
Acceptance Criteria:
  • Password reset link works correctly
  • Password meets all validation rules
  • Old password is verified before change
  • Confirmation email is sent

MEDIUM

User Profile Management

FR-UM-004.3: Additional Profile Fields (Recruiters):
  • Department

FR-UM-004.4: System shall auto-save changes
FR-UM-004.5: System shall validate all input fields
FR-UM-004.6: System shall show profile completion percentage

FR-JM-001.6: Job Status:

FR-JM-001.7: System shall auto-save draft every 30 seconds
FR-JM-001.8: System shall allow preview before publishing
FR-JM-001.9: System shall validate all required fields before publishing
FR-JM-001.10: System shall generate unique job ID (e.g., JOB-2026-001)

  • Draft saves automatically
  • Job publishes only when all required fields filled
  • Job ID is unique and auto-generated
  • Hiring manager receives notification

  • Cannot apply twice for same job
  • Draft save works correctly

MEDIUM

Withdraw Application

    • Salary expectations

  • Confirm withdrawal
FR-AM-003.3: Post-withdrawal:
  • Update status to "Withdrawn"
  • Send notification to recruiter
  • Cancel any scheduled interviews
  • Send confirmation email to candidate
  • Record withdrawal reason
  • Update application timestamp
FR-AM-003.4: Reapplication:
  • Allow reapplying after 30 days (configurable)
  • Show message if trying to reapply too soon
Acceptance Criteria:
  • Withdrawal works under correct conditions
  • Confirmation required before withdrawal
  • Recruiter is notified
  • Interviews are cancelled
  • Cannot withdraw when restrictions apply
